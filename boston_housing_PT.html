Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Dica:

  • R2 score com valor 0 significa que a varivel dependente no pode ser estimada pela varivel independente.
  • R2 score com valor 1 significa que a varivel dependente pode ser estimada pela varivel independente.
  • R2 score com valor entre 0 e 1 significa quanto a varivel dependente pode ser estimada pela varivel independente.
  • R2 score com valor 0.40 significa que 40 porcento da varincia em Y estimvel por X.

Resposta:

Os valores de R2 podem variar de 0 a 1, onde o valor 0 significa que a varivel dependente no pode ser estimada pela varivel independente.

No processo de obteno do Coeficiente de Determinao R2 iniciamos com uma linha que representa um "modelo errado", conhecida como baseline model, uma linha paralela ao eixo X. O baseline utilizar a mdia dos pontos que representam a varivel dependente para fazer a predio dos pontos que representam as variveis independentes. Ou seja, o baseline sea o ponto inicial de onde o modelo ser ajustado com base na mdia da distncia das variveis dependentes.

O R2 score obtido a partir do seguinte clculo:

$$R=1-\dfrac{SSE}{SST}$$

Onde, SSE representa a soma do quadrado dos erros do modelo de regresso e SST a soma dos quadrado dos erros do baseline. Ou seja, se os erros do modelo de regresso forem baixos o valor subtrado de 1 ser pequeno e o score R2 se manter prximo de 1.

Com base nas informaes acima podemos chegar a concluso de que esse modelo foi capaz de capturar a variao da varivel alvo com sucesso! Isso se deve ao fato de que o valor do Coeficiente de Determino obtido (R2), foi de 0.923 e isso mostra que mais de 90% da varincia da varivel alvo foi capturada pelo modelo.

Resposta:

A Validao Cruzada K-Fold uma tcnica de reciclagem de dados que consiste em dividir os dados em K partes/conjuntos, posteriormente treinando o modelo K vezes. Cada conjunto dividido em dois subconjuntos, o de treinamento e o de teste.

Essa tcnica permite que o modelo seja treinado e testado K vezes, e ao final das iteraes calcula-se a acurcia sobre os erros encontrados obtendo dessa maneira uma medida mais confivel referente ao modelo em questo.

Utilizar somente o modelo tradicional em que os dados so segregados em dados de treinamento e de teste, pode acarretar em resultados divergentes, principalmente quando estamos realizando uma busca em matriz. Situao na qual temos N modelos e precisamos test-los e trein-los com o mximo de variaes de dados possvel tanto para treinamento quanto para teste.

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parmetro the 'max_depth' para uma 
        rvore de deciso de regresso treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validao-cruzada para o treinamento de dados
    cv_sets = ShuffleSplit(n_splits=10, test_size = 0.20, random_state=0)

    # TODO: Gerar uma rvore de deciso de regresso de objeto
    regressor = DecisionTreeRegressor()

    # TODO: Gerar um dicionrio para o parmetro 'max_depth' com um alcance de 1 a 10
    params = {'max_depth': range(1,11)}

    # TODO: Transformar 'performance_metric' em uma funo de pontuao utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Gerar o objeto de busca em matriz
    grid = GridSearchCV(regressor, params, scoring_fnc, cv=cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo timo
    grid = grid.fit(X, y)

    # Devolver o modelo timo depois de realizar o ajuste dos dados
    return grid.best_estimator_
